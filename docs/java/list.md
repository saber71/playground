### 一、并发编程与多线程 (JUC)

*这部分是区分中级和高级开发者的分水岭，重点考察对锁机制、内存模型和线程池调优的理解。*

1. **Synchronized 与 ReentrantLock 的深层对比**
    * 除了基本用法，它们在底层实现（对象头标记 vs AQS队列）、锁升级过程（偏向锁->
      轻量级锁->重量级锁）、以及等待/通知机制（Object.wait/notify vs
      Condition）上有什么本质区别？
    * 在什么场景下你会优先选择 `ReentrantLock` 而不是 `synchronized`
      ？请结合公平锁和非公平锁的实现原理说明。

2. **volatile 关键字与 JMM (Java内存模型)**
    * `volatile` 如何保证可见性和禁止指令重排序？请结合内存屏障（Memory
      Barrier）原理解释。
    * `volatile` 能保证原子性吗？如果不能，为什么 `i++` 操作在多线程下不安全？如何利用
      `AtomicInteger` 或 `LongAdder` 解决高并发下的计数问题？（追问：`LongAdder`
      为什么比 `AtomicLong` 性能更好？）

3. **线程池 (ThreadPoolExecutor) 的生产级调优**
    * 请详细描述线程池的7个核心参数及其工作流程。当任务提交时，线程池是如何判断是创建新线程、放入队列还是执行拒绝策略的？
    * **场景题**：在一个CPU密集型任务和IO密集型任务混合的系统中，如何合理设置
      `corePoolSize` 和 `maximumPoolSize`？
    * 如果线上出现线程池队列堆积（OOM风险），你如何排查？动态调整线程池参数有哪些方案（如结合配置中心）？

4. **ThreadLocal 的原理与内存泄漏**
    * `ThreadLocal` 的底层数据结构是什么（ThreadLocalMap）？Key 为什么设计成弱引用？
    * 在什么情况下会发生内存泄漏？如何正确清理 `ThreadLocal`
      变量？在线程池复用线程的场景下，如果不清理会有什么后果？

5. **AQS (AbstractQueuedSynchronizer) 核心原理**
    * AQS 是如何实现资源共享和线程排队等待的？`state` 变量的作用是什么？
    * 请简述 `CountDownLatch`、`CyclicBarrier` 或 `Semaphore` 基于 AQS 的实现逻辑。

### 💡 备考建议

针对这些中高级问题，面试官通常不会只听你背诵定义，而是会采用 **"追问式"** 提问：

* "你刚才提到了AQS，那如果我在持有锁的时候发生了异常，AQS是怎么释放锁的？"
* "你说用了Redis做缓存，那如果Redis挂了，你的系统怎么保证不崩？"

建议在准备时，不仅要知道**是什么 (What)**，更要清楚**为什么 (Why)** 和 **怎么做 (
How)**，最好能结合自己项目中的实际案例（即使是模拟的）来阐述。
