网络模式有：bridge桥接、host主机、none无网络、overlay覆盖网络

## 桥接模式，默认模式
docker启动时会自动创建名为docker0的虚拟网桥
每个使用桥接模式的容器会被分配一个私有ip并通过veth pair连接到docker0
特点：隔离性好，每个容器又有独立网络命名空间；默认启用端口映射
适合单机开发、测试环境，多个容器需要在宿主机内部相互通信，但不需要直接暴露到宿主机网络

## 主机模式
容器直接共享宿主机的网络命名空间
容器不会获得独立ip，而是直接使用宿主机的ip和端口
特点：无NAT和虚拟网桥开销网络性能高，容器端口直接绑定到宿主机无需映射
但是安全隔离下差，端口冲突风险高
适合对网络性能要求极高的应用（如高性能代理、监控代理）、容器需要监听大量端口或特殊协议

## 无网络模式
容器拥有自己的网络命名空间，但不配置任何网络接口
完全断网，无法与外界或其他容器通信
特点：最大程度隔离、需要手动配置网络
适合执行离线任务（如数据处理、批处理作业）、安全敏感型任务禁止任何网络访问

## 覆盖网络模式
基于VXLAN技术在多个docker主机之间创建一个跨主机的虚拟网络
与docker swarm配合实现多节点容器通信
容器在overlay网络中拥有同一的子网ip
特点：支持跨主机通信、自动加密、服务发现、负载均衡
适合微服务架构中跨堕胎服务器的容器通信

## 自定义桥接网络
docker network create --driver bridge mynet
支持容器通过名称dns解析互相访问
更好的隔离性和网络配置控制
