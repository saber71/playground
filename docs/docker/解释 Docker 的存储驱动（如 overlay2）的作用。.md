存储驱动是docker管理容器和镜像文件系统的核心组件之一
主要作用：在宿主机上高效的实现镜像分写、写时复制以及容器可写层的机制，overlay2是docker默认且推荐的存储驱动

docker镜像由多个只读层组成，每层代表镜像构建过程中的一个指令。
当容器启动时，会在只读层之上加一个可写的顶层（称为容器层），所有对文件系统的修改都发生在整个容器层
存储驱动的作用就是将这些分层在宿主机文件系统上高效的组织和呈现为一个统一的文件系统视图共容器使用

overlay2基于linux的overlayFS文件系统实现，通过联合挂载技术将多个目录合并成一个统一的视图
lowerdir，对应镜像的多个只读层，多个按顺序堆叠
upperdir，容器的可写层，所有修改都发生在这里
merged，将lowerdir和upperdir联合挂载后呈现给容器的统一视图
workdir，OverlayFS内部使用的临时工作目录

overlay2优势
性能好，比早期的aufs和devicemapper更高效，尤其是在大量读写小文件时
支持多层，原生支持多层lowerdir
节省空间，利用写时复制机制，多个容器共享同一镜像时只读层只需存储一份
被内核广泛支持，稳定可靠
