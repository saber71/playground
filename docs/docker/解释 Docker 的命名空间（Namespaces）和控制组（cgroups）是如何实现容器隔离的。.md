# 命名空间，实现视图隔离
为每个容器提供一个独立的系统视图，使容器内的进程以为自己运行在一个独立的系统中，但实际上他们共享同一个内核

命名空间类型
PID，每个容器内的进程有独立的PID编号，与宿主机和其他容器隔离
NET，网络设备、IP地址、端口、路由表等，每个容器拥有自己的虚拟网络接口和IP
MNT，容器拥有独立的根文件系统，通过联合文件系统实现
UTS，主机名和域名，容器可以设置自己的hostname而不影响宿主机
IPC，信号量、消息队列、共享内存等IPC资源，容器内无法直接通过IPC通信
USER，支持将容器内的root映射到宿主机的非特权用户

# 控制组，实现资源限制和计量
用于限制、记录和隔离进程组使用的物理资源（CPU、内存、磁盘IO、网络带宽等）
主要功能有：资源限制、优先级分配、资源使用统计、任务控制（冻结、恢复一组进程）
Docker在启动容器时会为他创建对应的cgroup目录，写入资源限制参数
