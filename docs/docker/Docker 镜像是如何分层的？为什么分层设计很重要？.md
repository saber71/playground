如何分层？
镜像由多个只读层堆叠而成，每层对应Dockerfile中的一条指令（如RUN、COPY、ADD等）
每一层代表一次变更，记录了文件系统对于前一条的增量变化
docker使用联合文件系统将这些只读层叠加起来，形成一个统一的文件系统视图，最上层是可写的容器层，下面所有镜像层都是只读的
每一层通过其内容的SHA256哈希值唯一标识，相同内容的层在不同镜像间可共享节省存储空间

分层设计重要性？
利用层缓存可以加速构建，只要一层及其之前的层未改变，docker在镜像构建时就可以复用缓存
多个镜像如果共享相同的基础层，只需在磁盘上存储一份，减少硬盘占用
推送或拉取镜像时，docker只下载变化的层，而非整个镜像
每一层代表一次明确的变更，便于追踪镜像历史

注意事项
可以在单条RUN指令中用&&连接多个指令，避免产生不必要的中间层
在同一层安装软件后垃圾清理缓存后可以减少镜像体积
