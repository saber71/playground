开源的容器化平台，将应用程序与其依赖项打包到一个轻量级、可移植的容器中，实现一次构建、到处运行的模板

通过操作系统的内核特性（linux的命名空间和控制组cgroups）来隔离进程，让每个容器看起来像是在独立的环境中运行

虚拟机就像在一个大房子里再建一栋完整的、包括地基、水电、墙壁的小房子，每栋小房子都可以独立生活
docker就像在一个大房子里通过墙壁隔段划分出多个功能区，每个区域有自己独立的用途，但共有水电和地基（即宿主机的内核）

主要区别：
- 架构：docker共享宿主操作系统的内核；虚拟机运行独立完整的操作系统，而且可以与宿主机不同
- 启动速度：docker启动极快，秒级；虚拟机则通常需要几十秒到几分钟
- 资源占用：docker轻量，内存和cpu开销小；每个虚拟机都需要分配独立的资源
- 隔离性：docker进程隔离；虚拟机硬件级隔离
- 可移植性：docker可移植性极高，镜像可以在任何支持docker的系统上运行；虚拟机则受限于虚拟机平台
- 安全性：docker相对较低，共享内核的原因；虚拟机，高，因为是一个完全隔离的操作系统
- 典型用途：docker，微服务、CI/CD、开发测试环境；虚拟机，运行不同操作系统、强隔离需求场景
