物理复制，将预写日志WAL发送给备库，备库重放这些日志。备库的数据文件与主库在块级别上完全一致
优点
- 直接透传WAL，效率高，几乎没有解析和转换开销
- 配置简单，全库自动同步
- 强一致性保障，备库与主库完全一致
- 支持基于时间点的恢复
缺点
- 无法只同步几张表，缺乏灵活性
- 主备版本必须一致，升级时通常先升级备库
- 备库主要用于容灾和只读查询
- 无法直接将数据复制到其他数据库系统

逻辑复制，主库通过逻辑解码插件解析WAL日志，提取出具体的sql变更事件，然后通过网络发送给订阅端，订阅端重放这些sql
优点
- 很灵活，可以按需订阅特定的表
- 支持跨版本和跨平台
- 支持多主复制，但需要解决冲突
- 适合将数据同步到数据仓库、消息队列、搜索引擎或其他数据库
- 订阅端是一个独立的正常的数据库，可以同步的同时接收其他写入
缺点
- 逻辑解码和重放sql需要消耗更多cpu和内存
- 主库表结构修改时，备库需要手动调整表
- 如果备库被意外写入，可能导致主备数据库逻辑冲突，难以修复
- 对于大表的初始同步较慢
