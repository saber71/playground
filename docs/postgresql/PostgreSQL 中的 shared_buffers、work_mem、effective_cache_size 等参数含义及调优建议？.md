shared_buffers
pg管理的共享内存区域，用于缓存从硬盘读取的数据页（包括索引和表）。所有数据库会话共享这块内存
减少硬盘IO，提升查询性能，是pg的第一层缓存
默认值通常为128MB
调优：设为物理内存的15~25%
如果设置过高会挤压操作系统缓存，pg依赖系统缓存作为第二层缓存，写的过高，反而可能会降低性能

work_mem
每个数据库操作可使用的私有内存上限。一个复杂查询可能会使用多个work_mem
他决定了排序、去重、哈希连接等操作是否能在内存中完成。避免溢出到硬盘上的临时文件，可以大幅提升复杂查询性能
默认值通常为4MB
调优建议公式：总内存占用 ≈ shared_buffers + (max_connections × 平均并发操作数 × work_mem) + 其他
高并发简单查询：4MB~6MB
低并发复杂分析：64MB~256MB
混合：通过会话级动态调整或使用连接池限制实际并发

effective_cache_size
向查询优化器提示系统可用于缓存的总内存大小（shared_buffers + 操作系统文件系统缓存）。他并不是实际分配的内存
会影响优化器对执行计划的选择（如是否选择索引扫描或顺序扫描）
设置过低会导致优化器低估缓存能力从而选择更保守甚至更慢的计划
默认值通常为4GB
调优建议：设为物理内存的50%~75%
