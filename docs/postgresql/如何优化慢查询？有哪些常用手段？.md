诊断：
explain analyze + 真实sql语句，实际执行sql语句，返回真实的耗时和行数
explain buffers + 真实sql语句，显示共享缓冲区的命中情况，shared hit vs shared read，如果read数很高说明内存不够或索引无效发生了硬盘io
explain costs + 真实sql语句，显示优化器估算的sql性能

优化：
- 创建合适的索引，默认的B-Tree索引、复合索引、gin/gist
- select只取必要字段
- 避免在索引列上做函数计算避免索引失效
- 优化join，确保关联字段有索引，检查连接顺序
- 如无去重的需要，用union all代替union
- 分页优化，limit 100000，10 会很慢
- 对于超大表（千万/亿级），按时间或id进行声明式分区
- 对于复杂的报表查询，使用materialized view预先计算结果定期刷新
- 读写分离，将耗时的分析性查询路由到只读从库中
