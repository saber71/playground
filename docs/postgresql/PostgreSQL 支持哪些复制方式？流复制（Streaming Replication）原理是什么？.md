物理复制
- 流复制，主库将WAL日志实时传输给备库，备库重放这些日志来保持数据一致。
  - 同步流复制，主库提交事务前需等待至少一个备库反馈确认收到并写入日志，保证数据零丢失，但会增加延迟。
  - 异步流复制，主库提交事务无需等待备库，极端情况下可能会丢失数据
- 归档WAL文件，备库定期回放WAL日志，实时性较差，作为流复制的补充

逻辑复制，基于发布订阅模型，解析WAL日志中的逻辑变更将其转换为具体的sql操作或事件发送到订阅端
  支持表级复制，跨版本复制、跨平台复制，它不要求主从库的物理存储结构完全一致

流复制：
当备库请求复制时，主库启动专门的walsender后台进程，负责读取本地的WAL日志通过tcp连接发送给备库
备库启动时，运行walreceiver进程主动连接主库，它负责接收主库发来的WAL数据，将其写入本地的WAL文件
备库的startup进程负责读取并重放接收到的WAL日志
备库会定期主库发送反馈信息，告知主库自己已经接收和重放到了哪个位置
