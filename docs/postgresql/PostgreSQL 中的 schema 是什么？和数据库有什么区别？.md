是数据库对象（表、视图、函数、索引等）的逻辑容器或命名空间，一个数据库可以包含有多个schema，每个schema包含多个数据库对象

schema作用：
将相关的表、函数等组织在一起，便于管理
避免命名冲突
可以对schema设置权限，控制对其内对象的访问

区别：
- 数据库是一个独立的数据库实例；schema，数据库内部的逻辑分组
- 数据库之间完全隔离，不同数据库之间不能直接查询对方的表；schema，同一数据库下的不同schema可以通过指定schema互相访问
- 数据库连接时需指定数据库名；schema，数据库连接后切换或指定schema
- 每个数据库默认一个public的schema
- 数据库的创建有系统开销，因为他有一个独立的文件系统目录；schema的创建几乎无额外开销
- 数据库不能跨库查询；查询时指定同一数据库下的schema即可
